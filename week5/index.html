JavaScript: Novice to Ninja (Sitepoint)

Chapter 10: Testing and Debugging

  • Errors and bugs are a fact of life in programming
  • Three causes of errors: system error, programmer error, or user error
  • Interesting that a user error could be considered a programmer error because the site should be user friendly
  • Exceptions: an error that produces a return value that can then be used by the program to deal with the error (ex. something not defined)
  • Stack traces: a sequence of function or method calls that lead to the point where the error occured; traces all occurances of the error back to the original error so you can see where the error started.
  • Warnings: occurs if there is an error in the code that isn't enough to cause the program to crash.
  • put: 'use strict'; at top of JS to ensure you debug using strick mode and not sloppy
  • Linting tools (JS Lint, JS Hint, ES Lint) are useful with enforcing a programming style guide which is particularly useful when working in teams to make sure all team members are using the same conventions.
  • Debugging: the process of finding out where bugs occur in the code and then dealing with them.
  • Console.log is widely used to see what a variable is while debugging.
  • debugger keyword: stops the running of the code at certain breakpoints and you can hover over the variables and see their values
  • RangeError: thrown when number is outside allowable range of values
  • ReferenceError: thrown when a reference is made to an item that doesn't exist
  • SyntaxError: thrown when there's an error in the code's syntax
  • TypeError: thrown when there's an error in the type of value used (ex. string instead of number)
  • Can specify what error code should say in JS (just like in Python)
  • An exception causes the program to terminate with an error message (good for debuggin) however, it will cause a program to crash, not wanted in a final product
  • Testing: can simply be a function that tests if a piece of code runs as it should.
  • TDD: Test Driven Development; process of writing tests before any actual code
  • Jest is a built in TDD framework created by Facebook
  • Bugs are unavoidable and better to find early rather than later.
  • Exceptions can be thrown using the throw statement.
  • An error object is created when an exception occurs.
  • **Any code placed inside a try{} block will pass any error objects to a catch block when an exception occurs.
  • **Any code inside a finally {} block will run if an exception does or does not occur.
