Object Methods: this

  • Method: functions that are stored in object properties
  • OOP: Object-Oriented Programming
  • To access the object, a method can use the "this" keyword
  • Use "this" instead of just referencing the outer variable (name of the object) is to avoid errors
  • "This" is "free": can be used for multiple different objects when a function is called. By using "this", you can use functions like they are meant to be used, not just for one object at a time, but many.
  • Arrow functions don't have their own "this"; they take info from the outer "normal" function
  • The value of "this" is defined at run time
  • Ladder object allows you to do up and down with numbers

This in JavaScript

  • "this" in a global context defaults to the window ojbect, but this isn't usually used
  • "this" in an ojbect construction refers to the instance, but only when calling the function with "new" (ex. function Person(age) { this.age = age; } const greg = new Person(22);
  • Use "bind" to use "this" to reference the object in an event listener
  • "bind" doesn't work with arrow functions
  • "this" is a crucial keyword in JavaScript

Chapter 7: Events

  • Events: another part of the DOM; are what provides the link between the web page and user interactions. "Every time a user interacts with a web page, such as clicking on a link, pressing a key, or moving a mouse, an event occurs that our program can detect and then respond to.
  • Event Listeners: like setting a notification to alert you when something happens (when a user does something on the web page)
  • Every time a page is clicked, a call back function is called - this allows the rest of the program to run while waiting for something to be clicked
  • Click Event: works when mouse is clicked, enter is pressed, or screen is tapped
  • Event as a Parameter: many developers often abbreviate the word event to just an 'e'
  • Add an event listener to the whole web page by not putting document before the addEventListener
  • Use "document" first when you are trying to do things to the HTML, use a variable when trying to do things within the JS
  • Do not use anonymous functions as an argument to an addEventListener if you want to remove it later with removeEventListener (they need to both reference the same function
  • Think carefully before using the preventDefault() method because users expect certain things to happen and preventing them can cause confusion (like a link not going to the link when clicked
  • An event "propagates" as it moves from one element to another
  • Event Propagation: is the order that the evengs fire on each element; two kinds: bubbling and capturing
  • Bubbling: when event fires on element clicked on first then bubbles up the document tree, firing an even ton each parent element until it reaches the root node (usually the html)
  • Capturing: starts by firing an event on the root element, then propagates downward, firing an event on each child element until it reaches the targeted element that was clicked on
  • Event Delgation: used to attach an event listener to a parent element in order to capture events triggered by its child elements
